# Workspace Customization Rules for PinPulse Seeder

This document outlines the rules and strategies for database seeding in the PinPulse project. Future AI coding assistants must adhere to these directives when executing seeding requests.

---

## 1. Dual Seeding Strategy

The project utilizes two distinct database and data seeding paradigms:

### A. Hackathon Mock Database (Local In-Memory Cache)
- **Goal**: Allow the live demo to run instantly, offline, and without hitting API rate limits or incurring LLM costs.
- **Pipeline**: Ingests Excel spreadsheets (`pinpulse_youtube_seed.xlsx`) containing YouTube transcripts and video IDs.
- **Execution Script**: `backend/run_transcript_seeder.py`
- **Output**: Generates `backend/pinpulse_mock_db.json`.
- **Instruction**: When a team member uploads Excel spreadsheets or requests a mock seeder rebuild, immediately run this script. Do not attempt to query live search APIs.

### B. Live Real-Time Production Database (Supabase SQL)
- **Goal**: Fetch actual current live trends from social and local boutique signals.
- **Pipeline**: Calls Google Places API (Text Search) and YouTube Search API to retrieve active metadata, uses Gemini to match visual styles, generates embeddings, and uploads them directly to Supabase tables (`creators`, `creator_videos`, `regional_boutique_trends`).
- **Execution Script**: `backend/run_trend_extraction_pipeline.py`
- **Instruction**: Trigger this script only when the user requests a live synchronization using active API credentials (`GOOGLE_MAPS_API_KEY`, etc.).

---

## 2. AI Assistant Instructions
- **Proactive Recognition**: If a user uploads directories or Excel files, automatically inspect the schema. If they mention "seeding the transcripts" or "building the mock data," refer to this strategy and launch `backend/run_transcript_seeder.py`.
- **Model Consistency**: Always ensure that text vectors generated by either script match the 512-dimension size of the Myntra product catalog. Import `get_vibe_vector` from `backend/embed_catalog.py` to maintain mathematical parity.

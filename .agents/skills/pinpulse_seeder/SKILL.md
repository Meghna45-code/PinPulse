# Instruction for PinPulse Seeding Skill

You have triggered the automated seeding pipeline for regional fashion creators and local boutiques. When the user (or their team member) provides fashion transcripts, excel files, or requests to rebuild the databases, follow these steps:

## 1. Differentiate the Task
Determine if the request is for the **Mock Database** (hackathon caching) or the **Live Production Database**:
- **Mock Cache (`pinpulse_mock_db.json`)**: If they upload Excel sheets (like `pinpulse_youtube_seed.xlsx`) containing YouTube transcripts/video IDs.
- **Live Supabase SQL Database**: If they want to retrieve real-time data using the Google Places Places API and YouTube Search API.

## 2. Ingestion Workflows

### A. If seeding the Mock Cache:
1. Locate the input Excel file in the workspace root.
2. Run the transcript seeder:
   ```powershell
   python backend/run_transcript_seeder.py [path_to_excel]
   ```
3. Verify that `backend/pinpulse_mock_db.json` is updated.

### B. If seeding the Live Production Database:
1. Confirm that `GOOGLE_MAPS_API_KEY` and Supabase keys are configured in `backend/.env`.
2. Run the extraction pipeline:
   ```powershell
   python backend/run_trend_extraction_pipeline.py
   ```
3. Check the CLI output for Supabase row insert logs.

## 3. Verify Embedding Parity
All vectors generated by the seeder must be **512-dimensional** to match the Myntra catalog. Always use `get_vibe_vector` imported from `backend/embed_catalog.py`.
